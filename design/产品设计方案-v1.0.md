## 二、已确认的设计决策

| # | 决策项 | 结论 |
|---|--------|------|
| 1 | **年级覆盖** | 一到六年级，每级 25~30 词，共约 150+ 词 |
| 2 | **题型切换** | 首页选：仅填空 / 仅拖拽 / 混合（默认） |
| 3 | **自定义单词** | 只输入英文，AI 自动释义 → 缓存复用 |
| 4 | **进度管理** | 打卡制，记录连续天数，不重置 |
| 5 | **语音方案** | 浏览器 Web Speech API（后继 Tauri 版换 edge-tts 本地 mp3 缓存） |
| 6 | **填空模式提示** | 语音播报单词 + 右上角展示完整单词3秒 + 自动填入首空，该题得分减半 |
| 7 | **拖拽模式提示** | 语音播报首字母 + 自动填入首槽，该题不计分（0分） |
| 8 | **UI 风格** | 花园童话风 — 薄荷绿 + 樱花粉，卡通温馨 |
| 9 | **技术栈** | Phase 1: Web 原型 → Phase 2: Tauri Windows 桌面版 |
| 10 | **开发工具** | OpenCode + superpowers agent + deepseek-v4-pro |

## 三、页面结构（4个核心页面）

```
┌─────────────┐
│  ① 首页     │  ← 入口：年级选择 / 自定义单词 / 数量设置 / 题型选择
├─────────────┤
│  ② 测验页   │  ← 核心：场景一（点击填空）/ 场景二（拖拽拼词）
├─────────────┤
│  ③ 结果页   │  ← 每次测验结束：对错统计 + 错误清单 + 错题入口
├─────────────┤
│  ④ 历史/错题│  ← 历史记录列表 + 错题本复习入口
└─────────────┘
```

## 四、各页面设计概要

### ① 首页
- 出题方式：按年级出题（下拉选择年级）/ 自定义单词（英文输入，逗号分隔，AI 自动释义）
- 今日设置：单词数量（默认10，可+-调整）/ 题型选择（填空/拖拽/混合）
- 底部快照：今日已背 ?/10 | 连对 ? 天 | 错题本 ? 个待复习
- 入口按钮：开始测验 / 历史记录 / 复习错题

### ② 测验页
- 顶栏：进度（3/10）+ 得分 + 🔊重听按钮
- 场景一（填空）：挖空字母用虚线圈+脉冲动画 → 下方 4~6 个字母选项 → 点击填入 → 点击已填可撤回 → 提示按钮
- 场景二（拖拽）：上方槽位（虚线）→ 下方打乱字母方块（可拖拽）→ 拖到空白处 → 点击已填撤回 → 提示按钮
- 混合模式：奇数题用场景一，偶数题用场景二
- 答对：撒花 + 得分弹出 + 1.5秒后自动下一题

### ③ 结果页
- 本轮成绩：✅ 8/10，错了2个
- 错误单词清单（可点击复习）
- 按钮：复习错题 / 返回首页

### ④ 历史/错题页
- 历史记录：每次测验的日期、年级、正确率
- 错题本：每个错词出现次数，支持单独复习或全部复习

## 六、技术方案

| 层面 | Phase 1（Web原型） | Phase 2（Tauri桌面版） |
|------|:--:|:--:|
